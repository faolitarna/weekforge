# Plan 01-02: Raw Session Collection (Thin Tier-0)

## Goal

Build the thin data-collection layer: fetch Notion session pages, collect all block children
and comments into a structured `RawWeekData` bundle, compute checkbox arithmetic. Zero semantic
parsing. Zero LLM calls.

Spec: `specs/steps/step-1b-tier0-extraction.md` (revised ‚Äî thin Tier-0 approach)

## Context

- Step-1a complete: settings, loaders, CLI stub in place
- Tier-0 deliberately thin: session types vary (gym/climbing/hiking), comments are free-text
- All semantic extraction (params, roles, comment interpretation, delta) moves to step-1c LLM
- Decision rationale: `.planning/notes/tier0-thin-extraction.md`

## Threat Model

- Notion block schema changes ‚Üí all field access via `.get()`, never direct key access
- Empty session list ‚Üí `ValueError` with week prefix, never silent empty result
- Comment fetch failure ‚Üí log warning, continue with `comments=[]`, never crash

## Tasks

### Wave 1

- [ ] **Create `src/weekforge/models/week_summary.py`**
  - Define output contract: `Role`, `Status`, `ExerciseLogEntry`, `SessionLine`, `CardioEntry`,
    `ClimbingEntry`, `PainStatus`, `SectionRates`, `SkippedPattern`, `ImplicitFeedback`,
    `PlanAdherence`, `WeekSummary`
  - Match field types from spec exactly (stringly-typed weights, freeform reps)
  - This is the schema step-1c LLM fills ‚Äî Tier-0 only populates `implicit_feedback`

- [ ] **Create `src/weekforge/models/raw_week_data.py`**
  - `RawBlock` dataclass: `block_type`, `text` (plain string), `checked: bool | None`, `raw: dict`
  - `RawSession` dataclass: `page_id`, `name`, `blocks: list[RawBlock]`, `comments: list[str]`
  - `RawWeekData` dataclass: `week_prefix`, `sessions: list[RawSession]`, `planned_plan_markdown: str | None`

- [ ] **Create `src/weekforge/tools/raw_session_collector.py`**
  - `collect_blocks(page_id, notion_client) -> list[RawBlock]`
    - Fetch block children, flatten `rich_text` array to plain string per block
    - All field access via `.get()` ‚Äî never `block["key"]`
  - `collect_comments(page_id, notion_client) -> list[str]`
    - Fetch page comments, return plain-text bodies
    - On failure: log warning, return `[]`
  - `collect_raw_sessions(session_pages, notion_client) -> list[RawSession]`
    - One `RawSession` per page ‚Äî collect blocks + comments
    - Raise `ValueError` if `session_pages` is empty
  - `compute_checkbox_analysis(sessions) -> ImplicitFeedback`
    - Track `current_section` from `heading_2`/`heading_3` blocks
    - Bucket `to_do` blocks by section for `section_rates`
    - `frequently_skipped`: exercise where `skip_count / appearances > 0.5`
    - `always_completed`: exercise where `skip_count == 0` across all appearances
    - Pure arithmetic ‚Äî no interpretation
  - `assemble_raw_week(week_prefix, session_pages, notion_client, planned_plan_markdown) -> RawWeekData`
    - Calls `collect_raw_sessions`, raises `ValueError(f'{week_prefix}: no sessions found')` when empty

- [ ] **Create `tests/tools/test_raw_collector.py`** (inline dict fixtures, no fixture files)
  - `test_collect_blocks_extracts_text` ‚Äî to_do block with rich_text ‚Üí text field correct
  - `test_collect_blocks_checked_state` ‚Äî checked=True and False both captured
  - `test_compute_checkbox_analysis_counts` ‚Äî 3 sessions mixed checks ‚Üí correct totals
  - `test_compute_checkbox_analysis_section_rates` ‚Äî heading_2 sets section ‚Üí bucketing correct
  - `test_compute_checkbox_analysis_frequently_skipped` ‚Äî unchecked 2/3 ‚Üí in frequently_skipped
  - `test_assemble_raw_week_empty_raises` ‚Äî empty pages ‚Üí ValueError with week_prefix
